## Problem

Given an integer array nums sorted in non-decreasing order, remove the duplicates in-place such that each unique element appears only once. The relative order of the elements should be kept the same.

Consider the number of unique elements in nums to be k​​​​​​​​​​​​​​. After removing duplicates, return the number of unique elements k.

The first k elements of nums should contain the unique numbers in sorted order. The remaining elements beyond index k - 1 can be ignored.

```

Output: 2, nums = [1,2,_]
```

```

Output: 5, nums = [0,1,2,3,4,_,_,_,_,_]
```


## What I tried first

The first thing that comes to mind is that a new array can be created, which of course, is not a valid solution when it comes to in place sorting.
Moreover, the question asks only for the number of elements up to which every element is unique, which accounts for the fact that the remainder of the array can still remain a mess.

## Optimal approach

The solution involves the concept of two pointers .
The first pointer, ```i``` runs through every single element in the array and the condition for checking the duplicate(which obviously lies right before the pointer since the array is sorted).
If the condition isnt true(they're unequal) that's where the second pointer ```k``` comes in, and it helps to see where the unique element is to be placed.
Once the loop ends, the value of ```k``` is returned, which is the actual number of unique elements.


## Solution

```java
class Solution {
    public int removeDuplicates(int[] nums) {
        if(nums.length == 0)
        return 0;
        int k = 1;
        for(int i = 1; i < nums.length; i++){
            if(nums[i] != nums[i-1]){
                nums[k] = nums[i];
                k++;
            }  
        }
        return k;
    }
}
```
